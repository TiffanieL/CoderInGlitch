Description

Do you have a favorite recipe that you want to share with the world? With this project we will use code to write a recipe that looks as good as it tastes.

Step 2: Let's look around

Now we’re going to take a look at our new project. All Glitch projects have the same features so let’s take a second to familiarize ourselves with them.

Integrated Development Environment
When you begin a new project you can see the Glitch integrated development environment, or IDE for short. What does that mean? An IDE is a software application that allows you as the programmer to develop your software in a fluid way. Rather than switching between many programs, this has it all in one! A source code editor (which usually has autocomplete abilities), build tools (that compile your source code into that pretty final website!), and debugging tools (that shows you where errors are). What we’re looking at is the source code for our new web project, keep reading to learn about the parts of this IDE.

  • A. Project Button: This will display a menu where you can change the project name and description, find the live URL, create a new project or Remix this one, or switch to one of your other projects.
  • B. Show Live Button: Click this button to see your webpage in a new tab - this is a real web page you can have others visit!
  • C. Search Bar: This is where you can search for something in your files or assets, you'll use this as your project gets bigger.
  • D. User Options: This will open a menu that allows you to change your theme (the color of the development environment), and select or deselect options to see live changes on your website (if your internet is slow it may help to deselect this) and wrap text, or sign out.
  • E. Glitch options: This menu will allow you to open Community Project or the forum (where you can get help) in a new tab. There are other features in here that you can explore later on.
  • F. User controls: This icon will open up a menu that allows you to leave the project, and once other users join your project it will give you the option to thank them for help or remove them from the project.
  • G. Share: This menu gives you the option to share a link to view your code, and another link to invite others to join your project and code with you! You also have the option to make your project private, so that only users you have invited to join the project can see it.
  • H. Collapse: Click this button to collapse or expand this side-bar.
  • I. Logs: This opens an activity log at the bottom of the development environment. It allows you to see things like when people visit your website, what kind of browser they are using, and more. You'll check this out later on.
  • J. New File: Here you can add another file to create more web pages or other CSS and JavaScript files.
  • K. Assets: Here is where you can see assets or the things you are using in your project like images and sound files.
  • L. Remix this: This button will allow you to save a copy of the current file.
  • M. Markdown: This button switches between the Markdown editing or the live view. Markdown is a markup language, essentially it is a text-to-HTML tool that allows you to write simple commands in your text that gets displayed with formatting - like changing font size, bold, italics, bullet lists, etc. We will explore this more in another lesson.

Step 3: Where's the heading?

Now that we know our way around let’s start coding! All Glitch website projects start with a little bit of code already in them so let’s use what’s there and build on it.

  • In the development environment click on index.html. We’ll be doing a lot of our work here.
  • Take a look at the code. You might have noticed that the lines start with a bit of text that looks like this: “<word>”. This is called a tag. Tags are the bits of HTML that define the structure and content of a site. They are very important in HTML. Most tags have an open tag and a close tag. A close tag looks like this </tag>. See that slash? That’s what makes a closed tag close. Most tags need an opening and a closing or else they won’t work, you will learn about the ones that don't need closing tags (such as images) later on.
  • Find the <h1> tag. This is the tag for a heading. Right now the heading looks like this:
    <h1>Hi there!</h1>
  • Let’s make this the title of our recipe. Change it to the name of sandwich you want to write a recipe for. You can use whatever sandwich you like. We called ours "The Perfect PB&J". 
    <h1>The Perfect PB&J</h1>
  • Glitch auto-saves your work, and you should see it updating in your live preview. Notice how the headline text changed, but everything else stayed the same.

Step 4: Let’s start describing it

Now that we know how to change the content of tags, let’s change a few more and add some new ones to start putting together our recipe.

  • See the <p> tag underneath the <h1> we just changed? That’s a paragraph tag. We’re going to turn it into our recipe description.
    Right now the <p> tag looks like this:
    <p>I'm your cool new webpage. Made with 
<a href="https://www.glitch.com">Glitch</a></p>
  • Change it to instead describe your recipe.

Ours looks like this:

<p>This peanut butter and jelly sandwich is my favorite sandwich. It has the perfect balance of ingredients and looks great when made right.</p>
  • Your recipe description should now be on screen.
  • Up to this point, we’ve only modified tags that were already in the project. Let’s add a new one! We’ll add a small headline to our description.
  • Add a new line of code between the <h1> and <p> tag.
  • Now let’s add an <h2> tag. It’s a headline, like the <h1> but won’t be as prominent. The <h2> tag should read “Description.”

Your code should look like this:

<h1>The Perfect PB&J</h1>
<h2>Description</h2>
<p>This peanut butter and jelly sandwich is my favorite sandwich. It has the perfect balance of ingredients and looks great when made right.</p>

Step 5: One rule to divide them all

Now our recipe title and section title are butting into one another. That’s no good. Let’s add a line between them to keep them separate and divide the recipe into distinct sections.

Between the <h1> and <h2> tags make a new line and add a single <hr /> tag.

It should look like this:

<h1>The Perfect PB&J</h1>
<hr />
<h2>Description</h2>

Remember, all tags need to be closed. There are, however, some elements that are simple and don't contain content. These tags close themselves, using the slash (/) at the end of the tag.

The <hr /> tag is one of those tags. HR stands for “Horizontal Rule” and it’s an HTML element designed to divide content on a page.

Take a look. See how the <hr /> line separates the title from the description?

Step 6: What else do we need?

Any good recipe needs a list of ingredients. Fortunately HTML gives us an easy way to make list using tags.

  • Let’s start by creating a new section like the description section we’ve already made.
  • Make a new code line and add another <hr /> tag underneath our description.
  • Beneath that <hr /> add an <h2> tag for the headline “Ingredients.”

Our code looks like this:

<hr />
<h2>
  Ingredients
</h2>

Now let’s add a new type of tag called an “un-ordered list.” An un-ordered list tag produces a nice list of things, each with their own bullet point. In fact, you’re reading an un-ordered list right now!

Un-ordered lists actually use two tags. One starts and ends the list. It looks like this <ul>. The other sits in between those tags and identifies a list item. It looks like this <li>.

  • Make a new line of code and add an <ul> tag.
  • For each list item make a new line of code and enter an ingredient or tool needed for your recipe with a <li> tag.

Ours looks like this:

<h2>
  Ingredients
</h2>
<ul>
  <li>2 slices of white bread</li>
  <li>1 jar of grape jelly</li>
  <li>1 jar of smooth peanut butter</li>
  <li>A butter knife</li>
  <li>A sharp knife</li>
  <li>A cutting board</li>
</ul>

Step 7: How do we make it?

In the last step, we made an un-ordered list for our ingredients. Now let’s make an ordered list to describe each step we need to make our recipe. Ordered lists look like un-ordered lists except they have numbers instead of bullets.

  • Let’s start a new section just like the last one. We'll add the recipe directions beneath the “Ingredients” section.
  • Add an <hr /> tag and an <h2> tag with the headline “Directions.”
  • Start a new line and and add an <ol> tag. This is the tag for our ordered list.
  • Start a line underneath that and begin adding <li> tags for each direction step.
  • Don’t forget to close your <ol> tag at the end of the list with </ol>.

Our code looks like this:

<hr />
<h2>
  Directions
</h2>
<ol>
  <li>Lay both slices of bread next to each other on a cutting board.</li>
  <li>With the butter knife, spread a 1/8 inch layer of peanut butter on the left side</li>
  <li>On the right side, spread a 1/8 inch layer of jelly.</li>
  <li>Carefully place the two halves together so that the jelly is on top </li>
  <li>With the sharp knife, carefully trim the crust from the sandwich.</li>
  <li>Cut the sandwich diagonally.</li>
  <li>Enjoy the PB&J!</li>
</ol>

Step 8: What does it look like?

Our recipe has come together nicely so far. We have all the basic elements in place. What we need now is an image. So let’s add one. If you don’t have an image of your own you can use this one for now:

Drag it to your desktop or download it someplace where you can get access to it easily. We will be uploading it into Glitch soon.

  • Let’s put our image between our title and description sections.
  • Add a new line of code and a new <hr /> tag underneath the recipe title.
  • Between the two <hr /> tags, add a new line of code.
  • Add an <img src="" /> tag. It’s one of those tags that doesn’t need a close tag. Like <hr />.

It should look like this. Notice the src="" part. That's an attribute of the img tag that will let us specify the image we want to load.

<hr />
<img src="" />
<hr />
  • Now let’s go up to the top of our Glitch project. Remember the "Assets" section? It's above the index.html file. Click that to see your assets (or images, sound files, etc).
  • Click on the "Add Asset" button at the top, or you can drag and drop the file from the file explorer on your computer.
  • Select either the image we’ve provided that you already downloaded earlier or your own image.
  • Once the image is uploaded, you should see appear in the asset gallery.
  • Click on the image in your asset library, and then click on the button that says "Copy URL". You will see a notification near the top right that says "Link Copied!".
  • Now we’re going to add our image. Click off of the image window popup, and go back to the index.html file. Position your cursor in between the two quote symbols ("") in the <img src="" /> tag we created.
  • With the cursor still in that position, either right click and select Paste or use the keyboard commands to paste (Windows: "ctrl + v" or Mac: "cmd + v"). A bunch of text that looks a little bit like a website address should appear between the quotes. That's the path to your image file.

It should look something like this:

<img src="https://cdn.glitch.com/4b56d586-3f94-4a58-8971-ff3444b12293%2Fpbj_sandwich.jpg?1509224238469" />
  • Save your code and take a look at the preview version? Did the image appear?
  • If you added your own image, it might be too big or too small. You can shrink it down and change its size within the <img /> tag by adding height and width attributes.

Here's an example:

<img src="https://cdn.glitch.com/4b56d586-3f94-4a58-8971-ff3444b12293%2Fpbj_sandwich.jpg?1509224238469" height="400px" width="700px" />

Try adding height and width attributes to your <img /> tag. See what happens when you only use either the height or width attribute, but not the other. Did the image get squished or did it stretch evenly? Play around with different measurements until you like the way it looks on the page.

Step 9: Time for some style

So far we’ve only been adding new code to the HTML section. This is the content of our project, the text and images that make up our recipe. The text we’ve added, the headings, the lists, and the paragraphs all have a default style applied to them. The browser is picking the type, size, and color of the font. Let’s customize it the way we want using CSS.

  • Open the styles.css file now. CSS stands for “Cascading Style Sheets.” This file determines the style of the HTML elements, or the way they look. As you can tell there’s not much in here right now except a few bits of code that look like this:
h1 {
  color: fuchsia;
}

CSS is what we use to style HTML elements. Just like HTML tags, it needs to be written a specific way to work.

Generally, CSS styles look like this, keep an eye on those curly braces. They are very important and mark the start and finish of a style:

selector {
    property: value;
}
  • Let’s explore how this actually works. In the CSS file add a new line of code in the h1 style. Add the word padding for the property, and something huge like 300px for the value and then save the code.
  • Take a look at the preview now. See how the all the content shifted? That’s because we increased the padding, or space between the text and everything else.
  • CSS styles affect the way things look in the HTML. It’s in this file that will make our text look pretty by changing fonts and colors.
  • Let’s not leave our padding so big. Let’s change the style to only add 50px of padding to the left and 30px to the top sides. We can write styles that are more specific and apply multiple properties to a single selector.

h1 {
  color: fuchsia;
  padding-left: 50px;
  padding-top: 30px;
}

Step 10: Center content and change font family, style, and size

Now that we know a little about styles, let’s start writing our own. Let's begin by making our content centered.

  • Start a new line of code in the CSS tab.
  • For your selector all you need to do is write body and the CSS will change the style of the whole body tag in your HTML. After that, add an open curly brace, Glitch will automatically add the closing curly brace. Hit return to make room for your code.

Your code should look like this:

body {
  
}
  • On the empty line in between the curly braces let’s add the property width and let’s make the value 700px to match our picture. This will change the width of the content of your page to 700px, maximize your preview window to see the change.
body {
  width: 700px;
}
  • Notice how all of your content is now to one side. Let's fix that by giving it a margin value of auto. This looks a little better!
body {
  width: 700px;
  margin: auto;
}
  • Let’s change the style of the heading. Go ahead and add three more properties and values to the h1 style: font-family: courier, monospace; , font-size: 50px; , and font-weight: bold;
h1 {
  color: fuchsia;
  padding-left: 50px;
  padding-top: 30px;
  font-family: courier, monospace;
  font-size: 50px;
  font-weight: bold;
}

Take a look, the heading is now a different font, size, and bold!

Step 11: Colorize!

Let's mix up the color on the page to make it more interesting.

  • In the h1 style change the color value to red
  • The text changed to red didn’t it? Try another color name like yellow or black and see what happens.

You can use color names to apply colors in CSS. These are generic values, though and aren’t always exactly the color you want. If you want a little more control over color then you’ll need to use Hex values.

Hex values are codes that help a browser show color. They look like this: #FF0000. That’s the hex value for red. Hex codes are six digits long and contain only the number 0-9 and the letters A-F.

Here are some other example hex values:
#00FF00 - only green
#0000FF - only blue
#C0392B - softer red
#16A085 - softer green
#3498D8 - softer blue
#D977D4 - pink
#F1C40F - yellow
#F39C12 - orange
#FFFFFF - white
#000000 - black
#999999 - grey

Instead of a color name try adding a hex value into the color property and see what happens. Try changing the numbers as well to see what new colors you can create. You can also head over to Google and search for "color picker" which will bring up a color selection tool. This will display the Hex value for the color, as well as some others that you will learn about later. We've picked a nice blue color with a Hex value.

This is what our code looks like now:

h1 {
  color: #119be0;
  padding-left: 50px;
  padding-top: 30px;
  font-family: courier, monospace;
  font-size: 50px;
  font-weight: bold;
}

Step 12: Copy/Paste/Tweak

Now that we’ve styled our h1 we’ve got to style everything else. The good news is that we’ve actually finished most of the work. All we need to do now is copy the h1 style we’ve created and change a few things to create new styles for h2, p, and li. Let’s start with h2.

  • Let’s copy and paste the everything style from our h1 style.
  • Change the selector in this new style to h2.
  • Now let's remove the padding-top and change the value of the font-size to be something a little smaller, we’ve made ours 30px.

Our code looks like this:

h2 {
  color: #119be0;
  padding-left: 50px;
  font-family: courier, monospace;
  font-size: 30px;
  font-weight: bold;
}
  • Now take a look. See how it change all three headings with <h2> tags in the HTML? That’s the power of style sheets. You can set one style and have it affect multiple elements at once.
  • Now let’s do the same for <p> and <li> tags. Copy and paste the heading styles and make adjustments that make sense for those pieces of the recipe such as making the font-size a little smaller, changing the color, and removing the padding and font-weight property.

Congrats, you’ve finished your recipe! It’s nicely laid out and easy to read, isn’t it? Good thing, too. You’ve probably worked up an appetite with all this coding. Now you have an idea of what to make for lunch.
